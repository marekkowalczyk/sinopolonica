## Governing Principle

Polish has a rich inventory of **productive derivational suffixes** that
create new word categories. These suffixes carry clear, independent
semantic content — they pass the test of P01 ("what does this morpheme
mean?"). Under A1, they receive characters.

This is the most commonly violated principle during drafting. The
temptation is to leave derivational suffixes in Latin script because they
"feel like" endings. The test is rigorous: if the suffix changes the
*category or referent* of the word (not just its grammatical form), it
is semantic and receives a character.

## Productive Suffix Inventory

### Place suffixes: 舍 (-arnia, -alnia)

The suffix *-arnia* / *-alnia* means "place where X is done or kept."
It is fully productive and semantically transparent. Character: 舍
(establishment, house, lodging).

| 漢字-Polish | Polish | Decomposition |
|---|---|---|
| 冊舍 | księgarnia | book + place = bookshop |
| 焼舍 | piekarnia | bake + place = bakery |
| 珈琲舍 | kawiarnia | coffee + place = café |
| 肉舍 | masarnia | meat + place = butcher's |
| 犬舍 | psiarnia | dog + place = kennel |
| 煙舍 | palarnia | smoke + place = smoking room |
| 洗舍 | pralnia | wash + place = laundry |
| 糖舍 | cukiernia | sugar/sweets + place = confectionery |

Note: 冊舍 (księgarnia) contains **two** semantic morphemes — 冊
(book/volume) and 舍 (place). The earlier rendering *冊arni* violated
P01 by leaving the place-suffix in Latin script.

### Agent suffixes: 者 (-arz, -nik, -ca)

The suffix *-arz* / *-nik* / *-ca* means "person who does X" or
"person associated with X." Character: 者 (person, one who).

| 漢字-Polish | Polish | Decomposition |
|---|---|---|
| 焼者 | piekarz | bake + person = baker |
| 醫者 | lekarz | heal + person = doctor |
| 書者 | pisarz | write + person = writer |
| 勞者ik | pracownik | toil + person = worker |
| 導者ik | przewodnik | lead + person = guide |
| 讀者ik | czytelnik | read + person = reader |

Note: the additional *-ik* on 勞者ik is a secondary diminutive/variant
agent suffix — it stays in Latin as it does not add semantic content
beyond what 者 already provides.

### Abstract quality: 性 (-ość)

The suffix *-ość* creates abstract nouns from adjectives, meaning "the
quality of being X." Character: 性 (nature, quality, -ness).

| 漢字-Polish | Polish | Decomposition |
|---|---|---|
| 闇性 | ciemność | dark + quality = darkness |
| 明性 | jasność | bright + quality = brightness |
| 自由性 | wolność | free + quality = freedom |
| 獨性 | samotność | alone + quality = loneliness |
| 善性 | dobroć | good + quality = goodness |

### Domain/state: 域 (-stwo)

The suffix *-stwo* creates nouns denoting a domain, state, or collective
entity. Character: 域 (domain, realm).

| 漢字-Polish | Polish | Decomposition |
|---|---|---|
| 王域 | królestwo | king + domain = kingdom |
| 富域 | bogactwo | rich + domain = wealth |
| 國域 | państwo | state + domain = state/country |
| 兄弟域 | braterstwo | brother + domain = brotherhood |

### Instrument/tool: 器 (-nik as instrument, -ka as tool)

When *-nik* or *-ka* creates an instrument rather than an agent,
character: 器 (instrument, vessel, tool).

| 漢字-Polish | Polish | Decomposition |
|---|---|---|
| 示器 | wskaźnik | show + instrument = indicator |
| 計器 | licznik | count + instrument = meter |

## Compound Words (P16)

Polish compounds with transparent internal structure are written as
multi-character sequences. Each semantic morpheme gets its own character.

### Native compounds

| 漢字-Polish | Polish | Literal |
|---|---|---|
| 己行 | samochód | self-go |
| 善為者 | dobrodziej | good-do-person = benefactor |
| 大家 | wielmoża | great-house = magnate |

### Silent classifiers (P16, optional)

A written compound may include a classifier that adds semantic precision
beyond what the spoken word encodes. This is a script-level enrichment
— the spoken form is unchanged. Silent classifiers must be explicitly
marked in the lexicon with the notation [+clf].

| Standard | With classifier [+clf] | Polish |
|---|---|---|
| 己行 | 己行機 | samochód (self-go-machine) |

The decision to use or omit the classifier is a stylistic one. Everyday
writing omits it; technical or formal writing may include it for
precision.

## Interaction with Case Endings

When a derivational suffix receives a character, the case ending attaches
to the *last* character in the sequence, just as it attaches to the end
of the word in speech:

| 漢字-Polish | Polish | Analysis |
|---|---|---|
| 冊舍 | księgarnia | book-place-NOM (zero ending) |
| 冊舍i | księgarni | book-place-GEN |
| 冊舍ię | księgarnię | book-place-ACC |
| 冊舍ach | księgarniach | book-place-PL.LOC |

## Checklist for Practitioners

When rendering any Polish word, apply P01 recursively:

1. Identify the root. Assign a character.
2. Check for verbal/prepositional prefixes. Assign characters per P02.
3. Check for derivational suffixes. Test each with "what does this suffix
   mean?" If it has independent meaning, assign a character per P03.
4. Whatever remains is inflectional. Leave it in Latin script per P04.
5. Verify: does any Latin-script segment pass the meaning test? If yes,
   go back to step 1.
